Overview

How it works and how I'm doing it

Autotune is the most famous software for making a singer sound better. The technique it uses however is called pitch correction; it will be referred to as such from now on.

Imagine you're playing on newly-tuned piano at Carnegie Hall. Each note you play is perfect, the audience adores you. Suddenly, a string in the piano gives, and the piano is no longer in tune. Every few notes you hit the corresponding key, and your song is ruined. What happened?

In western music, every key on the piano represents a semi-tone, some tone which has been determined in-tune over the years. Each semi-tone has some pitch, or frequency, determining the semi tone. If the piano string doesn't vibrate at that frequency, it's out of tune. The same holds for a voice. If a singer sings slightly off from a semi-tone, they don't sound in tune.

This is where pitch correction comes in handy. With pitch correction, we split a singer's voice up into little windows of data. We then find the pitch of each window. Chances are, the pitch will be slightly off from some autotune. So we change the data a little, such that thead window does match a semi tone, and is thus in tune.

Pitch correction sounds great, but it can cost a lot of processing power. That's why I use an FPGA, or field-programmable gate array, in this project. In essence, the FPGA allows me to configure electronic components to do many computations at the same time. This allows me to output a singers voice fast enough that they can sing perfectly in real time.
